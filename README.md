# しいたけ栽培支援モニタリングシステム
家庭でのしいたけ栽培における環境管理を自動化・可視化するために開発したIoTシステムです。  
実用性を重視し、**異常の早期検知**と**外出先からの状況把握**に注力しました。  

## ■開発の背景
しいたけは温湿度の変化に敏感で、特に夏場の高温や乾燥は失敗に直結します。   
外出中も安心して栽培を続けられるよう、「現在の環境をリアルタイムに確認でき、異常があれば即座に通知を受け取れる仕組み」を構築しました。

## ■主な機能
- **リアルタイム環境監視**：センサーで取得したデータをGoogleスプレッドシートに記録。
- **異常検知・通知機能**：高温や乾燥などを検知した際、ブザー/LED、Discord Webhookを通じて即座に通知。
- **専用Webモニター**：最新10件のデータをモバイルで見やすく表示するHTML UIを実装。

## ■外部サービス連携
- **Nature Remoによる空調管理**：本システムでの監視に基づき、Nature Remoのオートメーション機能を活用してエアコンを制御。  
  **自作デバイスによる監視＋既存製品による制御**を組み合わせることで、低コストかつ高信頼な「保護環境」を実現しています。

## ■技術的な工夫と学習のポイント
- **直感的なステータス表示**：
  大量のデータが並ぶスプレッドシートをスマホで確認するのは困難でした。そこで、GASのHtmlServiceを利用し、必要な情報だけに絞ったモニター画面を自作しました。
- **1分単位のログによる「生存確認」**：
  配線の接触不良等による停止に備え、1分間隔でロギング。スプレッドシートが更新され続けていること自体を稼働確認として活用しています。
- **保守性を高める「動作ログ」の実装**：
  不安定な通信環境でも原因追及ができるよう、実行履歴やエラー内容を自動記録するwriteLog関数を実装し、デバッグの効率化を図りました。
- **セキュリティへの配慮**：
  GitHubでの公開を前提とし、Wi-Fi情報やAPIキーなどの機密情報はsecrets.hやスクリプトプロパティに分離。誤って公開されないよう環境を整えました。

## ■使用技術
- **ハードウェア**：ESP8266, DHT22
- **バックエンド**：Google Apps Script
- **データベース**：Google Spreadsheets
- **フロントエンド**：HTML, CSS, Javascript
- **外部サービス**：Discord API, Nature Remo
